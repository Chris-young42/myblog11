# Linux、curl、grep、SQL 系统学习计划

## 1. 总目标

本计划面向零基础或基础薄弱的学习者，目标不是简单记住命令，而是能在真实开发和运维场景中熟练使用 Linux 命令行、curl、grep 以及 SQL。

最终能力目标：

- 能独立使用 Linux 命令行完成文件、目录、日志、进程、网络等日常操作。
- 能熟练使用 `grep` 检索代码、日志和文本数据。
- 能完整掌握 `curl` 的常见和进阶用法，用于接口调试、问题复现和自动化脚本。
- 能写出可靠、清晰、高效的 SQL 查询。
- 能结合 `curl`、`grep`、`awk`、`sed`、`jq`、SQL 等工具解决真实开发和运维问题。

## 2. 学习原则

- 循序渐进：先掌握基础，再进入组合命令和真实场景。
- 不死记硬背：每个命令都要理解输入、输出、参数和适用场景。
- 以实战为主：每章必须包含命令示例、练习题和验收标准。
- 不胡编乱造：命令行为以官方文档、`man`、`--help` 和主流 GNU/Linux 环境为准。
- 明确差异：涉及 macOS/BSD、GNU/Linux、PostgreSQL、MySQL、SQLite 差异时单独说明。
- 可验证：所有关键命令都要求能在本机或测试环境中执行验证。

## 3. 技术基准

默认环境：

- 操作系统：GNU/Linux
- Shell：Bash
- HTTP 工具：curl
- 文本搜索：GNU grep
- SQL 主线：PostgreSQL

补充说明：

- macOS 默认命令多为 BSD 版本，部分参数和 GNU/Linux 不完全一致。
- MySQL、SQLite 和 PostgreSQL 的 SQL 语法大体相通，但函数、索引、执行计划、分页、时间处理等细节存在差异。
- 版本相关内容必须通过 `man 命令名`、`命令名 --help` 或官方文档确认。

## 4. 总体路线

学习分为六个阶段：

1. Linux 命令行基础
2. grep 全面掌握
3. curl 全语法体系
4. 开发和运维命令组合实战
5. SQL 基础到进阶
6. 综合项目训练

每个阶段都包含：

- 学习目标
- 核心知识点
- 必须掌握的命令或语法
- 常见错误
- 实战练习
- 验收标准

## 5. 阶段一：Linux 命令行基础

- 管道和重定向
- 环境变量
- 命令退出码

- 帮助系统

### 5.3 必须掌握的命令

文件和目录：

- `pwd`

- `mkdir`
- `touch`

- `mv`
- `rm`

查看文件：

- `cat`

系统辅助：

- `date`

- `id`
- `uname`

帮助查询：

管道和重定向：

- `|`
- `>`
- `>>`

- `2>>`

### 5.4 常见错误

- 不清楚当前目录，导致操作错文件。

- 混淆标准输出和标准错误。

### 5.5 实战练习

### 5.6 验收标准

- 能使用管道把多个命令连接起来。
- 能主动用 `man` 和 `--help` 查询命令用法。

## 6. 阶段二：grep 全面掌握

### 6.1 学习目标

掌握 `grep` 在文本、日志和代码检索中的核心用法，能用它快速定位关键内容。

### 6.2 核心知识点

- 普通文本匹配
- 大小写控制
- 行号显示
- 反向匹配
- 递归搜索
- 固定字符串匹配
- 单词匹配
- 正则表达式
- 上下文输出
- 多文件、多目录搜索
- 排除文件和目录

### 6.3 必须掌握的语法

基础搜索：

```bash
grep "error" app.log
```

忽略大小写：

```bash
grep -i "error" app.log
```

显示行号：

```bash
grep -n "error" app.log
```

反向匹配：

```bash
grep -v "debug" app.log
```

递归搜索：

```bash
grep -r "TODO" ./src
```

只显示匹配内容：

```bash
grep -o "user_id=[0-9]*" app.log
```

固定字符串匹配：

```bash
grep -F "a.b[c]" file.txt
```

扩展正则：

```bash
grep -E "error|failed|timeout" app.log
```

上下文：

```bash
grep -A 3 "Exception" app.log
grep -B 3 "Exception" app.log
grep -C 3 "Exception" app.log
```

排除目录：

```bash
grep -r "password" . --exclude-dir=.git --exclude-dir=node_modules
```

### 6.4 grep 正则重点

必须掌握：

- `.`：匹配任意单个字符
- `*`：匹配前一个字符零次或多次
- `^`：行首
- `$`：行尾
- `[]`：字符集合
- `[^]`：排除字符集合
- `+`：一次或多次，通常配合 `grep -E`
- `?`：零次或一次，通常配合 `grep -E`
- `|`：或者，通常配合 `grep -E`
- `()`：分组，通常配合 `grep -E`

### 6.5 常见错误

- 把 Shell 通配符和正则表达式混为一谈。
- 忘记给复杂模式加引号。
- 不知道 `grep`、`grep -E`、`grep -F` 的区别。
- 在大目录下递归搜索时没有排除 `.git`、`node_modules`、`target` 等目录。
- 用 `grep` 解析复杂结构化数据，而不是使用专门工具，例如 JSON 用 `jq`。

### 6.6 实战练习

1. 在日志中找出所有包含 `ERROR` 的行。
2. 找出所有非 `DEBUG` 的日志行。
3. 找出代码目录中所有 `TODO`。
4. 找出访问日志中所有 500 状态码。
5. 找出异常堆栈上下文。
6. 排除 `.git` 和 `node_modules` 后搜索关键字。

### 6.7 验收标准

完成本阶段后，应能做到：

- 能用 `grep` 快速定位日志和代码。
- 能写出基础正则表达式。
- 能区分普通匹配、扩展正则和固定字符串匹配。
- 能结合上下文参数分析错误现场。
- 能避免在大型目录中低效搜索。

## 7. 阶段三：curl 全语法体系

### 7.1 学习目标

掌握 `curl` 的 HTTP 请求、调试、认证、上传下载、代理、超时、重试、输出控制等能力，能用它复现和排查接口问题。

### 7.2 核心知识点

- URL
- HTTP 方法
- 请求头
- 请求体
- 响应头
- 响应体
- 状态码
- Cookie
- 认证
- HTTPS/TLS
- 代理
- 超时和重试
- 上传和下载
- 调试输出
- 脚本化调用

### 7.3 基础请求

GET 请求：

```bash
curl "https://example.com"
```

显示响应头：

```bash
curl -i "https://example.com"
```

只看响应头：

```bash
curl -I "https://example.com"
```

静默模式：

```bash
curl -s "https://example.com"
```

保存到文件：

```bash
curl -o index.html "https://example.com"
```

使用远程文件名保存：

```bash
curl -O "https://example.com/file.zip"
```

### 7.4 HTTP 方法

POST JSON：

```bash
curl -X POST "https://api.example.com/users" \
  -H "Content-Type: application/json" \
  -d '{"name":"alice","age":20}'
```

PUT：

```bash
curl -X PUT "https://api.example.com/users/1" \
  -H "Content-Type: application/json" \
  -d '{"name":"alice"}'
```

PATCH：

```bash
curl -X PATCH "https://api.example.com/users/1" \
  -H "Content-Type: application/json" \
  -d '{"age":21}'
```

DELETE：

```bash
curl -X DELETE "https://api.example.com/users/1"
```

注意：

- `-d` 默认会让 curl 使用 POST。
- 显式写 `-X POST` 有时不是必须的，但在教学和排查时可以增强可读性。

### 7.5 请求头和认证

设置请求头：

```bash
curl -H "Accept: application/json" "https://api.example.com/users"
```

Bearer Token：

```bash
curl -H "Authorization: Bearer TOKEN_VALUE" "https://api.example.com/me"
```

Basic Auth：

```bash
curl -u "username:password" "https://api.example.com/private"
```

Cookie：

```bash
curl -b "sessionid=abc123" "https://api.example.com/profile"
```

保存 Cookie：

```bash
curl -c cookies.txt "https://api.example.com/login"
```

读取 Cookie：

```bash
curl -b cookies.txt "https://api.example.com/profile"
```

### 7.6 表单和文件上传

普通表单：

```bash
curl -X POST "https://api.example.com/login" \
  -d "username=alice" \
  -d "password=secret"
```

multipart 文件上传：

```bash
curl -X POST "https://api.example.com/upload" \
  -F "file=@./report.pdf" \
  -F "description=monthly report"
```

从文件读取请求体：

```bash
curl -X POST "https://api.example.com/data" \
  -H "Content-Type: application/json" \
  -d @payload.json
```

### 7.7 调试和排错

详细输出：

```bash
curl -v "https://example.com"
```

更完整的 trace：

```bash
curl --trace trace.log "https://example.com"
```

ASCII trace：

```bash
curl --trace-ascii trace.txt "https://example.com"
```

输出状态码：

```bash
curl -s -o /dev/null -w "%{http_code}\n" "https://example.com"
```

输出耗时：

```bash
curl -s -o /dev/null -w "status=%{http_code} total=%{time_total}\n" "https://example.com"
```

### 7.8 超时、重试、重定向

连接超时：

```bash
curl --connect-timeout 3 "https://example.com"
```

总超时：

```bash
curl --max-time 10 "https://example.com"
```

失败重试：

```bash
curl --retry 3 "https://example.com"
```

跟随重定向：

```bash
curl -L "https://example.com"
```

### 7.9 HTTPS、证书和 TLS

跳过证书校验：

```bash
curl -k "https://self-signed.example.com"
```

指定 CA 证书：

```bash
curl --cacert ca.crt "https://internal.example.com"
```

注意：

- `-k` 只适合测试和临时排查，不应作为生产脚本的默认选项。
- 证书问题应该优先修复证书链、域名和信任配置。

### 7.10 代理、DNS 和 Host

HTTP 代理：

```bash
curl -x "http://127.0.0.1:8080" "https://example.com"
```

指定 Host 头：

```bash
curl -H "Host: api.example.com" "http://127.0.0.1:8080"
```

指定域名解析：

```bash
curl --resolve "api.example.com:443:10.0.0.5" "https://api.example.com"
```

### 7.11 curl 常见错误

- JSON 字符串引号写错。
- 忘记设置 `Content-Type: application/json`。
- 使用 `-k` 掩盖真实证书问题。
- 只看响应体，不看状态码和响应头。
- 没有设置超时，导致脚本长时间卡住。
- 把 token、密码直接写入可提交的脚本。

### 7.12 实战练习

1. 用 `curl` 请求一个 GET 接口并保存响应。
2. 用 `curl` 提交 JSON。
3. 用 `curl` 带 token 调接口。
4. 用 `curl -v` 分析请求头和响应头。
5. 用 `curl -w` 输出状态码和耗时。
6. 使用 `--connect-timeout` 和 `--max-time` 控制超时。
7. 上传一个文件。
8. 使用 Cookie 模拟登录后的请求。

### 7.13 验收标准

完成本阶段后，应能做到：

- 能用 `curl` 构造常见 HTTP 请求。
- 能清楚区分请求头、请求体、响应头、响应体。
- 能用 `curl -v` 和 `--trace` 排查接口问题。
- 能处理认证、Cookie、上传、下载、代理、证书等场景。
- 能把 `curl` 命令写入脚本并控制超时和失败行为。

## 8. 阶段四：开发和运维命令组合实战

### 8.1 学习目标

把单个命令组合起来，解决真实开发和运维问题。

### 8.2 核心组合

查看实时错误日志：

```bash
tail -f app.log | grep --line-buffered "ERROR"
```

统计错误数量：

```bash
grep "ERROR" app.log | wc -l
```

统计访问日志中的状态码：

```bash
awk '{print $9}' access.log | sort | uniq -c | sort -nr
```

统计访问最多的 IP：

```bash
awk '{print $1}' access.log | sort | uniq -c | sort -nr | head
```

搜索代码但排除依赖目录：

```bash
grep -r "TODO" . --exclude-dir=.git --exclude-dir=node_modules
```

批量查找文件：

```bash
find . -name "*.log"
```

批量搜索：

```bash
find . -name "*.log" -print0 | xargs -0 grep "ERROR"
```

接口健康检查：

```bash
curl -s -o /dev/null -w "%{http_code} %{time_total}\n" "https://example.com/health"
```

JSON 格式化：

```bash
curl -s "https://api.example.com/users/1" | jq .
```

### 8.3 实战场景

场景一：线上接口报 500

处理流程：

1. 用 `grep` 在日志中搜索接口路径。
2. 用 `grep -C` 查看错误上下文。
3. 用 `awk` 统计错误发生时间段。
4. 用 `curl` 复现接口请求。
5. 用 `curl -v` 检查请求和响应细节。
6. 用 SQL 查询相关业务数据。

场景二：接口变慢

处理流程：

1. 用访问日志统计接口耗时。
2. 找出最慢的请求。
3. 用 `curl -w` 测试接口响应时间。
4. 用 SQL `EXPLAIN` 查看查询计划。
5. 判断是否需要加索引或优化 SQL。

场景三：批量检查服务状态

处理流程：

1. 准备 URL 列表。
2. 使用循环和 `curl` 检查 HTTP 状态码。
3. 输出失败的地址。
4. 记录响应时间。

### 8.4 验收标准

完成本阶段后，应能做到：

- 能熟练使用管道组合命令。
- 能从日志中定位错误。
- 能统计状态码、IP、接口路径和错误数量。
- 能用 `curl` 配合日志复现问题。
- 能把常见排查流程整理成脚本。

## 9. 阶段五：SQL 基础到进阶

### 9.1 学习目标

掌握 SQL 的查询、修改、聚合、多表关联、事务、索引、执行计划和性能优化。

### 9.2 数据模型基础

必须理解：

- 数据库
- Schema
- 表
- 行
- 列
- 主键
- 外键
- 索引
- 约束
- 事务

### 9.3 基础查询

查询全部列：

```sql
SELECT * FROM users;
```

查询指定列：

```sql
SELECT id, name, email FROM users;
```

条件查询：

```sql
SELECT * FROM users WHERE status = 'active';
```

排序：

```sql
SELECT * FROM users ORDER BY created_at DESC;
```

分页：

```sql
SELECT * FROM users ORDER BY id LIMIT 20 OFFSET 40;
```

### 9.4 聚合查询

计数：

```sql
SELECT COUNT(*) FROM orders;
```

分组：

```sql
SELECT status, COUNT(*)
FROM orders
GROUP BY status;
```

分组过滤：

```sql
SELECT user_id, COUNT(*) AS order_count
FROM orders
GROUP BY user_id
HAVING COUNT(*) >= 5;
```

常用聚合函数：

- `COUNT`
- `SUM`
- `AVG`
- `MAX`
- `MIN`

### 9.5 数据修改

插入：

```sql
INSERT INTO users (name, email)
VALUES ('alice', 'alice@example.com');
```

更新：

```sql
UPDATE users
SET status = 'inactive'
WHERE id = 1;
```

删除：

```sql
DELETE FROM users
WHERE id = 1;
```

注意：

- 执行 `UPDATE` 和 `DELETE` 前必须确认 `WHERE` 条件。
- 生产环境修改数据前应先 `SELECT` 验证影响范围。

### 9.6 多表查询

INNER JOIN：

```sql
SELECT users.name, orders.id, orders.amount
FROM users
INNER JOIN orders ON orders.user_id = users.id;
```

LEFT JOIN：

```sql
SELECT users.name, orders.id
FROM users
LEFT JOIN orders ON orders.user_id = users.id;
```

### 9.7 子查询和 CTE

子查询：

```sql
SELECT *
FROM users
WHERE id IN (
  SELECT user_id
  FROM orders
  WHERE amount > 100
);
```

CTE：

```sql
WITH high_value_orders AS (
  SELECT *
  FROM orders
  WHERE amount > 100
)
SELECT user_id, COUNT(*)
FROM high_value_orders
GROUP BY user_id;
```

### 9.8 窗口函数

排名：

```sql
SELECT
  user_id,
  amount,
  ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY amount DESC) AS rn
FROM orders;
```

累计求和：

```sql
SELECT
  user_id,
  created_at,
  amount,
  SUM(amount) OVER (PARTITION BY user_id ORDER BY created_at) AS running_total
FROM orders;
```

### 9.9 事务

基本事务：

```sql
BEGIN;

UPDATE accounts
SET balance = balance - 100
WHERE id = 1;

UPDATE accounts
SET balance = balance + 100
WHERE id = 2;

COMMIT;
```

回滚：

```sql
BEGIN;

UPDATE users
SET status = 'inactive'
WHERE id = 1;

ROLLBACK;
```

必须理解：

- 原子性
- 一致性
- 隔离性
- 持久性
- 锁
- 死锁
- 隔离级别

### 9.10 索引和执行计划

创建索引：

```sql
CREATE INDEX idx_users_email ON users (email);
```

查看执行计划：

```sql
EXPLAIN SELECT * FROM users WHERE email = 'alice@example.com';
```

PostgreSQL 查看实际执行：

```sql
EXPLAIN ANALYZE SELECT * FROM users WHERE email = 'alice@example.com';
```

必须理解：

- 全表扫描
- 索引扫描
- 复合索引
- 最左前缀思想
- 选择性
- 回表
- 排序
- Join 策略

### 9.11 SQL 常见错误

- 使用 `SELECT *` 查询大量无用列。
- 忘记 `WHERE` 导致全表更新或删除。
- 在索引列上使用函数导致索引失效。
- 不理解 `NULL` 的比较规则。
- 混淆 `WHERE` 和 `HAVING`。
- 不理解 `INNER JOIN` 和 `LEFT JOIN` 的区别。
- 分页深度过大导致性能下降。
- 只看 SQL 语句，不看执行计划。

### 9.12 实战练习

1. 设计用户表、订单表、商品表。
2. 查询每个用户的订单数。
3. 查询最近 7 天订单总额。
4. 找出消费金额最高的 10 个用户。
5. 找出没有下单的用户。
6. 使用窗口函数找出每个用户金额最高的一笔订单。
7. 给高频查询字段添加索引。
8. 使用 `EXPLAIN` 分析索引是否生效。

### 9.13 验收标准

完成本阶段后，应能做到：

- 能写出常见业务查询 SQL。
- 能正确使用聚合、分组、多表关联。
- 能理解事务的基本作用。
- 能读懂基础执行计划。
- 能识别常见 SQL 性能问题。

验收：

### 10.3 综合项目三：SQL 报表

目标：

- 设计基础业务表。
- 插入测试数据。

### 10.4 综合项目四：从日志到数据的完整排查

目标：

1. 用户反馈接口报错。
2. 用 `grep` 从日志中定位错误。
3. 用 `curl` 复现接口。
4. 用 SQL 查询相关数据。
5. 判断是参数问题、数据问题、权限问题还是服务问题。
6. 输出排查报告。

验收：

- 能形成完整排查链路。
- 能把命令、输出、结论写清楚。
